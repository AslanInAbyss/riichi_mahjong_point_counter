本函數是一個基於 mahjong 套件封裝的高級算分引擎，支援複雜字串解析、符數計算、役種判定、明暗槓區分及役滿封頂限制。

函數point_count(input_str, allow_multi_yakuman=True, enable_kuitan=True, disable_aka_dora=False):
   
    input_str: 符合特定語法的牌局字串
    
    allow_multi_yakuman: 是否允許雙倍/複合役滿 (True: 累加, False: 32000/48000 封頂)
    
    enable_kuitan: 是否允許食斷 (預設 True)
    
    disable_aka_dora: 是否禁用赤寶牌 (預設 False, 即 0 視為赤 5)
    
    回傳: [是否成功, 役種列表, 役滿倍數, 總翻數, 符數, 總點數, 文字描述]
    
    
字串語法規則字串以 + 號作為分隔符，結構如下：[手牌區] + [和了牌/副露] + [Dora區] + [額外標籤/風位]
1. 手牌區 (Hand)格式：數字 + 花色後綴 (m, p, s, z)。字牌 (z)：1-7z 分別代表 東南西北白發中。赤寶牌：用 0 表示（如 0m 為赤 5 萬）。注意：手牌區僅包含「門前」的部分。
2. 副露與和了牌 (Melds & Win Tile)榮和：+1s (獨立的一個 + 後跟一張牌)。自摸：不需要額外寫，函數會自動取手牌區最後一張。吃/碰：+123m, +555z (長度為 3)。明、加槓：+1111m (長度為 4)。暗槓：+a1111m 或 +c1111m (加前綴 a 或 c)。
3. 寶牌區 (Dora)語法：+d 後跟寶牌指示牌（注意：是指示牌，不是寶牌本身）。範例：+d12z 代表指示牌為東、南，寶牌為南、西。
4. 額外標籤與風位 (Extras & Winds)
   標籤字母：
     r: 立直 (Riichi)
   
     i: 一發 (Ippatsu)
   
     w: 雙立直 (Double Riichi)
   
     t: 天和 / 地和 (Tenhou/Chiihou)
   
     h: 海底摸月 / 河底撈魚 (Haitei/Houtei)
   
     k: 嶺上開花 / 槍槓 (Rinshan/Chankan)
   
   風位設定：接在標籤後的兩位數字。1, 2, 3, 4 對應 東, 南, 西, 北。第一位為 場風，第二位為 自風（預設為 12 東場南家）。範例：+ri12 (立直一發，東場南家)、+22 (無標籤，南場南家)。

範例參考表說明字串範例:

    自摸 (門清)'123456789m1112s2s'
    榮和 (一發寶牌)'123456789m111s2s+2s+d1s6z+ri'
    副露 (吃碰)'123m55z+456m+666z'
    暗槓 (四暗刻)'11133m666z99s3m+a6666p'
    特定風位 (南場北家)'123m...z+34'
    ⚠️ 注意事項規則衝突防呆：若出現規則矛盾（如：副露後立直、海底與槍槓同時出現），函數會回傳 False 並附帶錯誤訊息。
封頂邏輯：當 allow_multi_yakuman=False 時，任何超過 13 翻或複合役滿的點數都會被強制鎖定在 32000（閒家）或 48000（莊家）。
